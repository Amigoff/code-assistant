# DroneManager JSON API - внешнее взаимодействие

## 1. Назначение

Интерфейс предназначен для обмена данными между системой управления (внешнее приложение) и средой моделирования:

- **Команды управления**: внешнее приложение отправляет управляющие воздействия для выбранного дрона.
- **Телеметрия**: среда моделирования отправляет агрегированную телеметрию по всем активным дронам.
- **Конфигурация**: внешнее приложение может обновлять сетевые параметры и список дронов во время работы.

Протокол является «сообщениями JSON поверх TCP/UDP» и не использует HTTP.

---

## 2. Роли и каналы связи

Термины, используемые ниже:

- **Сервер** - сторона моделирования (принимает конфигурацию и команды, отправляет телеметрию).
- **Клиент** - внешнее приложение (отправляет конфигурацию и команды, принимает телеметрию).

Каналы:

1) **TCP: Конфигурация**
- Сервер слушает TCP-порт `ReceivePort`.
- Клиент устанавливает TCP-соединение и отправляет **один JSON-объект конфигурации**.

2) **UDP: Команды**
- Сервер слушает UDP-порт `ReceivePort`.
- Клиент отправляет UDP-датаграммы с JSON-командами на этот порт.

3) **UDP: Телеметрия**
- Сервер отправляет UDP-датаграммы с телеметрией на `RemoteIP:RemotePort`.
- Клиент должен слушать `RemotePort` на адресе `RemoteIP` (либо на всех интерфейсах, в зависимости от сетевой схемы).

---

## 3. Кодировка и формат сообщений

- Кодировка JSON: **UTF-8**.
- Каждая UDP-датаграмма содержит **ровно один JSON-объект**.
- Для TCP-конфигурации рекомендуется:
  - отправлять **ровно один JSON-объект** в рамках соединения;
  - после отправки **закрывать соединение** со стороны клиента.

Ограничения UDP:
- Практически рекомендуется держать размер UDP-сообщения в пределах MTU (например, до ~1400 байт полезной нагрузки), чтобы избежать фрагментации. Для больших сообщений требуется учитывать риски потерь и фрагментации на маршруте.

---

## 4. Объекты данных

### 4.1. Vector3

Используется для 3D-вектора:

```json
{ "X": 0.0, "Y": 0.0, "Z": 0.0 }
```

Типы полей: числа (integer или float; принимаются как JSON number).

### 4.2. Quaternion

Используется для кватерниона:

```json
{ "X": 0.0, "Y": 0.0, "Z": 0.0, "W": 1.0 }
```

### 4.3. Transform

Используется для положения/ориентации/масштаба:

```json
{
  "Rotation":   { "X": 0.0, "Y": 0.0, "Z": 0.0, "W": 1.0 },
  "Translation":{ "X": 0.0, "Y": 0.0, "Z": 0.0 },
  "Scale3D":    { "X": 1.0, "Y": 1.0, "Z": 1.0 }
}
```

### 4.4. GeographicCoordinates

Используется для географических координат (широта/долгота/высота):

```json
{ "Latitude": 47.0, "Longitude": 38.9, "Altitude": 120.0 }
```

---

## 5. Конфигурация (TCP)

### 5.1. Назначение

Конфигурация определяет:
- куда отправлять телеметрию (`RemoteIP`, `RemotePort`);
- на каком порту принимать конфигурацию и команды (`ReceivePort`);
- список дронов (их идентификаторы и стартовые координаты).

### 5.2. Схема объекта конфигурации

```json
{
  "RemoteIP": "127.0.0.1",
  "RemotePort": 12345,
  "ReceivePort": 12346,
  "Drones": [
    { "ID": "Drone_1", "Pos": { "X": 0, "Y": 0, "Z": 500 } }
  ]
}
```

Поля:

- `RemoteIP` (string) - IPv4-адрес назначения для телеметрии (формат `a.b.c.d`).
- `RemotePort` (int) - UDP-порт назначения для телеметрии.
- `ReceivePort` (int) - порт, который сервер использует:
  - как TCP-порт для приёма конфигурации;
  - как UDP-порт для приёма команд.
- `Drones` (array) - список описаний дронов.

Элемент `Drones[]`:

- `ID` (string) - уникальный идентификатор дрона.
- `Pos` (Vector3) - стартовая позиция в географических координатах:
  - `X` - широта (Latitude),
  - `Y` - долгота (Longitude),
  - `Z` - высота (Altitude).

### 5.3. Рекомендованный сценарий применения

- Клиент устанавливает TCP-соединение на `ReceivePort` сервера.
- Отправляет JSON-объект конфигурации.
- Закрывает соединение.

Конфигурация может отправляться повторно для изменения параметров во время работы.

---

## 6. Команды управления (UDP)

### 6.1. Назначение

Команда задаёт состояние «armed» и значения по осям управления для конкретного дрона.

### 6.2. Схема команды

```json
{
  "id": "Drone_1",
  "armed": true,
  "controls": {
    "pitch": 0.0,
    "roll": 0.0,
    "yaw": 0.0,
    "throttle": 0.5
  }
}
```

Поля:

- `id` (string) - идентификатор дрона (должен совпадать с одним из `Drones[].ID` текущей конфигурации).
- `armed` (bool) - желаемое состояние armed.
- `controls` (object) - оси управления:
  - `pitch` (number)
  - `roll` (number)
  - `yaw` (number)
  - `throttle` (number)

Диапазоны и единицы измерения для осей управления должны быть согласованы между клиентом и сервером (в протоколе не фиксируются).

### 6.3. Поведение при ошибках

- Команды с неизвестным `id` игнорируются.
- Если JSON некорректен или отсутствуют обязательные поля, сообщение игнорируется.

---

## 7. Телеметрия (UDP)

### 7.1. Назначение

Телеметрия отправляется пакетами и содержит данные по всем активным дронам.

### 7.2. Схема телеметрии

Корневой объект:

```json
{
  "Drones": {
    "Drone_1": { /* FDroneTelemetryShort */ },
    "Drone_2": { /* FDroneTelemetryShort */ }
  }
}
```

Где ключи объекта `Drones` - это идентификаторы дронов.

### 7.3. Структура FDroneTelemetryShort

```json
{
  "Roll": 0.0,
  "Pitch": 0.0,
  "Yaw": 0.0,
  "Throttle": 0.5,

  "bArmed": true,

  "WorldTransform": {
    "Rotation": { "X": 0.0, "Y": 0.0, "Z": 0.0, "W": 1.0 },
    "Translation": { "X": 0.0, "Y": 0.0, "Z": 0.0 },
    "Scale3D": { "X": 1.0, "Y": 1.0, "Z": 1.0 }
  },

  "LinVel":  { "X": 0.0, "Y": 0.0, "Z": 0.0 },
  "AngVel":  { "X": 0.0, "Y": 0.0, "Z": 0.0 },
  "LinAcc":  { "X": 0.0, "Y": 0.0, "Z": 0.0 },
  "AngAcc":  { "X": 0.0, "Y": 0.0, "Z": 0.0 },

  "LinVelB": { "X": 0.0, "Y": 0.0, "Z": 0.0 },
  "AngVelB": { "X": 0.0, "Y": 0.0, "Z": 0.0 },
  "LinAccB": { "X": 0.0, "Y": 0.0, "Z": 0.0 },
  "AngAccB": { "X": 0.0, "Y": 0.0, "Z": 0.0 },

  "GPSCoords": { "Latitude": 47.0, "Longitude": 38.9, "Altitude": 120.0 },

  "bIsOnGround": false
}
```

Поля:

- `Roll`, `Pitch`, `Yaw`, `Throttle` (number) - агрегированное состояние органов управления (может использоваться для мониторинга).
  - `Throttle` может иметь значение `-1`, если данные отсутствуют/не инициализированы.
- `bArmed` (bool) - состояние armed.
- `WorldTransform` (Transform) - положение/ориентация/масштаб в мировой системе координат.
- `LinVel`, `AngVel`, `LinAcc`, `AngAcc` (Vector3) - линейная/угловая скорость и ускорение (мировая система координат).
- `LinVelB`, `AngVelB`, `LinAccB`, `AngAccB` (Vector3) - те же величины в локальной (body) системе координат.
- `GPSCoords` (GeographicCoordinates или Vector3) - географические координаты.
- `bIsOnGround` (bool) - признак нахождения на земле.

---

## 8. Минимальная последовательность обмена

1) Клиент отправляет конфигурацию по TCP на порт `ReceivePort` сервера.
2) Клиент начинает:
   - слушать UDP-порт `RemotePort` (приём телеметрии),
   - отправлять UDP-команды на `ReceivePort` сервера.
3) Сервер отправляет телеметрию UDP на `RemoteIP:RemotePort`.

---

## 9. Примеры

### 9.1. Конфигурация (TCP payload)

```json
{
  "RemoteIP": "127.0.0.1",
  "RemotePort": 12345,
  "ReceivePort": 12346,
  "Drones": [
    { "ID": "Drone_1", "Pos": { "X": 47.236, "Y": 38.896, "Z": 120.0 } },
    { "ID": "Drone_2", "Pos": { "X": 47.236, "Y": 38.897, "Z": 120.0 } }
  ]
}
```

### 9.2. Команда (UDP datagram)

```json
{
  "id": "Drone_1",
  "armed": true,
  "controls": { "pitch": 0.05, "roll": -0.02, "yaw": 0.1, "throttle": 0.6 }
}
```

### 9.3. Телеметрия (UDP datagram)

```json
{
  "Drones": {
    "Drone_1": {
      "Roll": 0.0,
      "Pitch": 0.0,
      "Yaw": 0.0,
      "Throttle": 0.5,
      "bArmed": true,
      "WorldTransform": {
        "Rotation": { "X": 0.0, "Y": 0.0, "Z": 0.0, "W": 1.0 },
        "Translation": { "X": 10.0, "Y": 0.0, "Z": 2.0 },
        "Scale3D": { "X": 1.0, "Y": 1.0, "Z": 1.0 }
      },
      "LinVel":  { "X": 1.0, "Y": 0.0, "Z": 0.0 },
      "AngVel":  { "X": 0.0, "Y": 0.0, "Z": 0.1 },
      "LinAcc":  { "X": 0.0, "Y": 0.0, "Z": 0.0 },
      "AngAcc":  { "X": 0.0, "Y": 0.0, "Z": 0.0 },
      "LinVelB": { "X": 1.0, "Y": 0.0, "Z": 0.0 },
      "AngVelB": { "X": 0.0, "Y": 0.0, "Z": 0.1 },
      "LinAccB": { "X": 0.0, "Y": 0.0, "Z": 0.0 },
      "AngAccB": { "X": 0.0, "Y": 0.0, "Z": 0.0 },
      "GPSCoords": { "Latitude": 47.236, "Longitude": 38.896, "Altitude": 120.0 },
      "bIsOnGround": false
    }
  }
}
```
