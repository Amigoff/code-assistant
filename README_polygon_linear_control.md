# polygon_linear_control.py — Справочник параметров

Скрипт управляет роем дронов в симуляторе Polygon с использованием алгоритма GCBF+.

---

## Быстрый старт

```bash
python polygon_linear_control.py \
  --model-path "logs/LinearDrone/gcbf+/seed0_..." \
  --leader-id Drone_5 \
  --formation wedge --form-before-fly \
  --waypoints "6,0,0 6,6,0 0,6,0 0,0,0" \
  --ctrl-scale 0.6 --slow-radius 3.0 \
  --pos-scale 500.0 --invert-pitch \
  --takeoff-alt 1.0 --takeoff-throttle 0.5 \
  --fix-throttle --base-throttle 0.0 \
  --area-size 15.0
```

---

## Параметры

### Сеть и порты

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--server-ip` | `127.0.0.1` | IP симулятора |
| `--receive-port` | `12346` | UDP-порт симулятора для приёма команд (ReceivePort) |
| `--telemetry-bind-ip` | `0.0.0.0` | IP для приёма телеметрии |
| `--telemetry-port` | `12345` | UDP-порт телеметрии (RemotePort) |
| `--send-interval` | `0.008` | Пауза (с) между пакетами к каждому дрону. Увеличить до 0.02–0.03, если летят не все |

---

### Модель GCBF+

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--model-path` | `None` | Путь к обученной модели (папка с `config.yaml` и `models/`). Если не задан — используется `u_ref` (LQR) |
| `--model-step` | `None` | Номер шага чекпоинта. `None` = последний |
| `--drone-config` | `../DroneConfig.json` | Путь к JSON-файлу со списком ID дронов |

---

### Среда GCBF+

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--area-size` | `5.0` | Размер рабочей области (ед. GCBF+). Должен соответствовать обучению |
| `--num-agents` | `None` | Кол-во агентов. По умолчанию берётся из DroneConfig.json |
| `--obs` | `0` | Кол-во случайных препятствий |
| `--max-step` | `256` | Макс. шагов эпизода |
| `--max-travel` | `None` | Макс. дистанция перемещения |
| `--max-seconds` | `None` | Остановить управление через N секунд |
| `--seed` | `1234` | Зерно генератора случайных чисел |
| `--cpu` | флаг | Запустить JAX на CPU (без GPU) |
| `--control-dt` | `0.05` | Период цикла управления (с). `0.05` = 20 Гц |

---

### Масштабирование координат

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--pos-scale` | `1.0` | Делитель позиций: `pos_gcbf = pos_sim / pos_scale`. Если дроны стоят с шагом 500 ед. и GCBF+ обучен на area_size=5, используйте `100` |
| `--vel-scale` | `= pos-scale` | Делитель скоростей. По умолчанию равен `pos-scale`. Изменить если скорости клипуются (диапазон GCBF+: ±1.5 ед/с) |

---

### Безопасные зоны GCBF+

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--drone-radius` | `0.2` | Радиус безопасной зоны (ед. GCBF+). Минимальное расстояние между дронами = 2×drone_radius |
| `--comm-radius` | `1.0` | Радиус связи: дроны видят соседей только внутри этого радиуса. Должен быть ≥ шага формации |

---

### Лидер и формация

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--leader-id` | `Drone_1` | ID дрона-лидера (например `Drone_5`) |
| `--formation` | `grid` | Тип формации: `grid` — решётка, `wedge` — клин |
| `--follower-z-offset` | `0.0` | Смещение высоты ведомых относительно лидера (ед. GCBF+). `-0.5` = ведомые на 0.5 ниже |

**Параметры решётки** (только `--formation grid`):

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--grid-step-x` | `5.0` | Шаг решётки по X (используется только для warmup) |
| `--grid-step-y` | `5.0` | Шаг решётки по Y |

**Параметры клина** (только `--formation wedge`):

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--wedge-dx` | `1.0` | Расстояние между рядами клина вдоль оси движения (ед. GCBF+) |
| `--wedge-dy` | `1.0` | Расширение крыльев клина на каждый ряд (ед. GCBF+) |
| `--yaw-gain` | `0.5` | Коэффициент P-регулятора yaw. Больше = резче поворот |
| `--yaw-rate` | `2.0` | Макс. скорость поворота формации (рад/с) |

**Схема клина для 9 дронов** (`--wedge-dx 1.0 --wedge-dy 1.0`):
```
            [Лидер]
         [D]       [D]        ряд 1: ±(1.0, 1.0)
      [D]               [D]   ряд 2: ±(2.0, 2.0)
   [D]                     [D] ряд 3: ±(3.0, 3.0)
[D]                           [D] ряд 4: ±(4.0, 4.0)
```

---

### Фаза построения клина

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--form-before-fly` | флаг | Сначала построить клин на месте, потом лететь к точкам |
| `--form-ctrl-scale` | `0.15` | Масштаб управления при построении клина (медленно) |
| `--form-threshold` | `0.3` | Максимальная ошибка позиции ведомых (ед. GCBF+) при которой считается что строй построен |

---

### Маршрутные точки

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--waypoints` | `None` | Список точек через пробел, каждая `dx,dy,dz` от стартовой позиции лидера. Пример: `"6,0,0 6,6,0 0,6,0"` |
| `--target-dx` | `5.0` | Смещение по X если `--waypoints` не задан |
| `--target-dy` | `0.0` | Смещение по Y |
| `--target-dz` | `0.0` | Смещение по Z |
| `--waypoint-threshold` | `0.5` | Расстояние (ед. GCBF+) при котором переключиться на следующую точку |
| `--waypoint-speed-threshold` | `0.15` | Макс. скорость лидера при которой разрешён переход к следующей точке |
| `--waypoint-loop` | флаг | Зациклить маршрут (вернуться к первой точке после последней) |

---

### Скорость и управление

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--ctrl-scale` | `1.0` | Масштаб pitch/roll (0..1). Максимальная скорость вдали от цели |
| `--slow-radius` | `0.0` | Расстояние (ед. GCBF+) с которого начинается плавное замедление. `0` = выключено |
| `--min-ctrl-scale` | `0.05` | Минимальная скорость у точки назначения (при `--slow-radius > 0`) |

**График адаптивной скорости** (`--ctrl-scale 0.8 --slow-radius 3.0 --min-ctrl-scale 0.05`):
```
ctrl_scale
  0.8 |████████████\
      |             ████\
  0.4 |                  ███\
      |                      ██\
 0.05 |                         ────
      └─────────────────────────────→ dist до цели
      3.0 (slow-radius)              0
```

---

### Взлёт

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--takeoff-alt` | `0.0` | Высота взлёта (ед. GCBF+). `0` = пропустить фазу взлёта |
| `--takeoff-throttle` | `0.65` | Газ при взлёте |
| `--takeoff-timeout` | `120.0` | Таймаут взлёта (с) |

---

### Throttle

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--fix-throttle` | флаг | Держать throttle = base-throttle постоянным (игнорировать az из GCBF+). Рекомендуется |
| `--base-throttle` | `0.5` | Базовый газ в фазе GCBF+. `0.5` ≈ висение. Подбирать по дрону |
| `--throttle-gain` | `0.25` | Коэффициент для az: `throttle = base_throttle + gain*az` (только без `--fix-throttle`) |

---

### Ориентация осей управления

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--invert-pitch` | флаг | Инвертировать знак pitch. Включить если дрон летит назад вместо вперёд |
| `--invert-roll` | флаг | Инвертировать знак roll |
| `--controls-keys-upper` | флаг | Отправлять `Pitch/Roll/Throttle` с заглавной вместо `pitch/roll/throttle` |

---

### Торможение и демпфирование

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--brake-radius` | `0.0` | Радиус активного торможения (ед. GCBF+). Внутри зоны действие заменяется на `-brake_gain×velocity`. `0` = выключено |
| `--brake-gain` | `2.0` | Сила тормоза. Больше = резче остановка. Типичные значения: 1.0–5.0 |
| `--vel-damp` | `0.0` | Фоновое демпфирование: вычитает `kd×velocity` из ускорения. Подавляет колебания. Типичные значения: 0.2–0.8 |
| `--repulsion-radius` | `0.0` | Радиус явного межбортового отталкивания (ед. GCBF+). `0` = выключено |
| `--repulsion-gain` | `1.0` | Сила отталкивания. Типичные значения: 0.5–3.0 |

---

### Помехи канала связи

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--cmd-delay` | `0.0` | Базовая задержка команд (с). Все пакеты задерживаются минимум на это время |
| `--cmd-jitter` | `0.0` | Макс. случайный джиттер сверх базовой задержки (с). Каждый пакет: равномерно [0, jitter] |
| `--drop-prob` | `0.0` | Вероятность потери пакета (0.0–1.0). Каждый дрон независимо |

**Пример комбо**: `--cmd-delay 0.3 --cmd-jitter 0.7 --drop-prob 0.1` →
задержка 0.3–1.0 с, 10% пакетов теряется.

| Параметр | По умолчанию | Описание |
|---|---|---|
| `--verbose-commands` | флаг | Выводить JSON и pitch/roll/throttle для каждого дрона на каждом шаге |

## Фазы выполнения

```
1. [TAKEOFF]  Все дроны набирают высоту --takeoff-alt
              Газ = --takeoff-throttle

2. [FORM]     (только --formation wedge --form-before-fly)
              Лидер висит на месте, ведомые занимают позиции клина
              Скорость = --form-ctrl-scale
              Завершается когда max_err < --form-threshold

3. [NAV]      Весь строй летит по --waypoints
              Скорость адаптируется по --slow-radius
              Переход к следующей точке:
                dist < --waypoint-threshold И speed < --waypoint-speed-threshold
```

## Типичные команды

**Решётка, одна точка:**
```bash
python polygon_linear_control.py \
  --model-path "logs/LinearDrone/gcbf+/seed0_..." \
  --target-dx 6.0 --ctrl-scale 0.4 \
  --pos-scale 500.0 --invert-pitch \
  --takeoff-alt 1.0 --fix-throttle --base-throttle 0.0
```

**Клин, маршрут, с задержкой канала:**
```bash
python polygon_linear_control.py \
  --model-path "logs/LinearDrone/gcbf+/seed0_..." \
  --leader-id Drone_5 \
  --formation wedge --form-before-fly \
  --wedge-dx 1.0 --wedge-dy 1.0 \
  --waypoints "6,0,0 6,6,0 0,6,0 0,0,0" \
  --waypoint-threshold 0.5 --waypoint-speed-threshold 0.15 \
  --ctrl-scale 0.8 --slow-radius 3.0 --min-ctrl-scale 0.05 \
  --brake-radius 2.0 --brake-gain 3.0 --vel-damp 0.3 \
  --cmd-delay 0.5 --cmd-jitter 0.5 --drop-prob 0.05 \
  --pos-scale 500.0 --invert-pitch \
  --takeoff-alt 1.0 --takeoff-throttle 0.5 \
  --fix-throttle --base-throttle 0.0 \
  --area-size 15.0 --drone-radius 0.2 --comm-radius 3.0
```
